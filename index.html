Fundamentos de Docker

Entiende la idea antes de memorizar comandos.

¿Qué problema resuelve?

Docker empaqueta una aplicación con sus dependencias para que se ejecute igual en desarrollo, pruebas y producción. El contenedor comparte el kernel del sistema, por eso suele ser más ligero que una máquina virtual.

Imagen

Contenedor

Docker vs máquina virtual

Una máquina virtual incluye un sistema operativo completo. Un contenedor comparte el kernel del host y aísla procesos, red y archivos, por eso suele iniciar más rápido y consumir menos recursos.

Errores comunes

  • Puerto ocupado: cambia el puerto izquierdo, por ejemplo 8081:80.
  • Nombre repetido: borra el contenedor anterior con docker rm.
  • Datos perdidos: usa volúmenes cuando la información debe persistir.

Dockerfile generado

Docker Compose

Levanta varios servicios con un solo archivo YAML.

Ejemplo: app + base de datos

compose.yaml 
services:
  web:
    build: .
    ports:
      - "3000:3000"
    environment:
      DATABASE_URL: postgres://app:secret@db:5432/app
    depends_on:
      - db

  db:
    image: postgres:16-alpine
    environment:
      POSTGRES_USER: app
      POSTGRES_PASSWORD: secret
      POSTGRES_DB: app
    volumes:
      - db_data:/var/lib/postgresql/data

volumes:
  db_data:

Red automática

Los servicios se encuentran por nombre. En este ejemplo, `web` se conecta al host `db`.

Ciclo de trabajo

docker compose up --build

docker compose logs -f web

docker compose down

Cuándo usarlo

Es ideal para entornos locales con varias piezas: API, base de datos, cache, worker y herramientas de administración.
